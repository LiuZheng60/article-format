# article-format-lz · 自媒体文章排版生成技能

> 把「写好文案」到「可发布排版」压缩成一句话指令的 WorkBuddy 技能。
> 支持 **微信公众号**（内联 CSS）与 **今日头条**（语义 HTML）双平台，内置 **23 种风格** 与 **23 + 17 套排版组件**。

[![Styles](https://img.shields.io/badge/styles-23-green)](#风格总览)
[![License: MIT](https://img.shields.io/badge/license-MIT-yellow)](#许可证)

---

## ✨ 功能特性

- **双平台自动适配**：微信公众号采用全内联 `style` 确保复制不丢格式；今日头条采用纯语义 HTML 适配 ProseMirror schema 白名单。
- **23 种风格**：7 通用 + 8 节日 + 4 季节 + 4 主题，令牌化配色，换色即换风格。
- **全套组件**：微信 23 个组件（标题、列表、引用、提示框、数据卡、对比栏、代码块、表格、进度条、信息卡片、键值列表等）；头条 17 个语义组件。
- **智能风格匹配**：可由用户指定，也可根据节日 / 季节 / 主题关键词或内容类型自动选风格。
- **复制即所见即所得**：所有兼容性规则（如微信 `position:absolute`、`box-shadow` 实测支持）均基于真实编辑器粘贴验证，而非经验臆测。
- **宽表自适应**：宽表 / 长内容可自动改用信息卡片或键值列表展示，避免移动端溢出。

## 🎯 解决什么痛点

| 痛点 | 本技能的解法 |
|------|-------------|
| **复制丢失**：编辑器里调好的样式，粘到公众号后台全崩 | 微信全程内联 CSS，头条用语义标签，复制粘贴零损耗 |
| **双平台分裂**：微信和头条排版规则完全不同，写两遍 | 一套技能双引擎，根据平台产出不同 HTML |
| **配色纠结**：选什么主色、辅色、浅底永远拿不准 | 23 种已校准风格，按内容 / 节日自动匹配 |
| **组件拼装慢**：卡片、提示框、进度条手写半天 | 23 组件模板开箱即用，一句话触发 |
| **兼容盲猜**：`position:absolute`、`box-shadow` 到底行不行 | 全部基于实测写入，文档与生成逻辑一致 |

将本仓库的 `article-format-lz/` 目录复制到 WorkBuddy 的技能目录之一即可，**无需额外依赖**：

```bash

```

重启 / 刷新 WorkBuddy 后，在对话中直接下达排版指令即可触发（见下方「快速开始」）。

> 技能目录约定详见 WorkBuddy 文档；用户级与项目级技能均可被自动发现与调用。

## 🚀 快速开始

**示例 1 · 公众号排版（指定风格）**

```
用「墨韵素雅」风格排版这篇公众号文章：[粘贴文章]
```

**示例 2 · 节日自动匹配**

```
帮我排一篇关于端午龙舟的公众号推文
# 技能会自动命中「端午·艾绿五彩」风格
```

**示例 3 · 今日头条**

```
把这篇科技资讯排成今日头条格式，发头条用
```

```
同一篇内容，分别生成公众号和头条两个版本
```

## 🧩 排版组件

### 微信公众号（23 个，全内联 CSS）

> 使用 `<section>` 嵌套，所有样式内联，不用 class。组件编号 1–23，详见 `references/design-system.md`。

主标题区 · 分割线 · 导语引用块 · 二级标题 · 三级标题 · 正文段落 · 行内强调 · 强调卡片 · 无序列表 · 有序列表 · 数据高亮卡片 · 信息提示框 · 对比双栏 · 表格 · 进度条 · 信息卡片 · 引用块 · 键值列表 · 代码块 · 要点回顾 · 底部版权区 等。

### 今日头条（17 个，纯语义 HTML）

> 零 inline style，仅用语义标签 + Unicode 符号（◆ ★ ▶ 【】）建立层级，适配 ProseMirror 白名单。编号 T1–T17。

主标题 · 摘要 · 分割线 · 正文 · 加粗强调 · 章节标题 · 子标题 · 核心结论 · 无序列表 · 有序列表 · 引用块 · 数据高亮 · 表格 · 代码块 · 底部签名 · 信息卡片 · 键值列表。

## ⚙️ 工作原理

两套平台的渲染机制不同，因此生成逻辑完全分开：

| 维度 | 微信公众号 | 今日头条 |
|------|-----------|----------|
| 渲染机制 | 接受内联 CSS | ProseMirror schema 白名单，剥离 inline style |
| HTML 形态 | `<section>` + `style=""` | 纯语义标签（h1/h2/p/ul/blockquote…） |
| 层级建立 | 颜色 + 字号 + 装饰竖条 | Unicode 符号（◆★▶【】） |
| 代码块 | `<section>` + `<br>` + `&nbsp;`（禁 `<pre>`+white-space） | `<pre><code>`（schema 保留） |
| 配色 | 23 种可选风格 | 固定头条红 `#F04142` + 灰阶 |
| 兼容要点 | `position:absolute` / `box-shadow` 实测支持；`border-left` 更简洁 | 禁止 inline style 与 `<section>` 嵌套 |
